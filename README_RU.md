# Техническая документация ADB Studio Standalone (V12.0) [ОБНОВЛЕНО]

## 1. Обзор проекта
**ADB Studio** — это мощное автономное приложение для Android, предназначенное для расширенного управления устройствами, аудита безопасности и оптимизации системы. Начиная с версии 12.0, проект перешел на **Standalone Edition** (автономную версию), что исключает необходимость использования ПК или внешнего сервера.

## 2. Основная архитектура
- **Фреймворк:** React с TypeScript (Frontend) + Capacitor (Native Bridge).
- **Нативная реализация:** Jetpack Compose (Kotlin) для специализированных экранов.
- **Интеграция сервисов:** **Shizuku API**. Это ядро автономности V12.0, позволяющее приложению выполнять команды уровня ADB (`pm`, `am`, `du`, `shell`) напрямую на устройстве с правами оболочки.
- **Система плагинов:** Пользовательский плагин Capacitor `ShizukuExecutor` связывает веб-интерфейс с системной службой Shizuku.

## 3. Ключевые функции
### A. Менеджер приложений (APPS)
- **Категоризация:** Автоматическая группировка приложений по разделам: `ИСПОЛЬЗУЕМЫЕ` (Running), `ЗАМОРОЖЕННЫЕ` (Sleeping) и `ОТКЛЮЧЕННЫЕ` (Disabled).
- **Управление:** Кнопки для `STOP` (принудительная остановка), `OFF/ON` (отключение/включение) и `DEL` (удаление).
- **Групповые действия:** Выбор нескольких пакетов для выполнения массовых операций через динамическую панель Batch Action.
- **Оптимизированный UI:** Ручное обновление списка для экономии ресурсов и подробные описания состояний приложений.

### B. Запуск активностей (Activity Launcher)
- **Анализ компонентов:** Разбор данных `dumpsys package` для вывода всех экспортируемых и скрытых активностей любого приложения.
- **Мгновенный запуск:** Выполнение команды `am start -n` прямо из списка для запуска скрытых функций.

### C. Файловый менеджер (FS)
- **Навигация:** Просмотр локального хранилища, включая защищенные области, доступные через шелл.
- **Аудит памяти:** Расчет размеров директорий с помощью команды `du`.
- **Инспектор файлов:** Чтение и просмотр текстовых конфигураций или логов с помощью `cat`.

### D. AI Ассистент (Интеграция с Gemini) — НОВОЕ
- **Связь с API:** Подключение к Google Gemini API (модели Flash/Flash-Lite) через персональный API-ключ.
- **Аудит безопасности:** Автоматический аудит устройства в один клик (сбор параметров сборки, состояния батареи и разрешений) с последующим анализом уровня риска (Низкий/Средний/Высокий/Критический).
- **Анализ логов:** Использование ИИ для поиска индикаторов компрометации (IOC) и объяснения поведения приложений.
- **Экспорт отчетов:** Возможность сохранения истории чатов и отчетов аудита в `/sdcard/ADB_Studio_Logs/`.

### E. Исполнитель скриптов (Scripts) — НОВОЕ
- **Поддержка Bash и Python:** Запуск shell-скриптов или кода Python 3 напрямую через Shizuku.
- **Быстрые скрипты:** Встроенные шаблоны: "Ultimate Control V3.9", "Security Audit V2" и "Clean All Cache" (очистка кэша всех приложений).
- **Локальное хранилище:** Сохранение и экспорт пользовательских скриптов в память устройства.

### F. Терминал (Local Shell) — НОВОЕ
- **Интерактивный шелл:** Прямой интерфейс командной строки для выполнения любых ADB shell команд.
- **Руководство по командам:** Встроенная "шпаргалка" по продвинутым командам (например, оптимизация графики Element X, сброс настроек экрана).

### G. Резервное копирование (Backup)
- **Массовый бэкап:** Извлечение APK установленных приложений в папку `/sdcard/Download/ADB_Backups/`.
- **Отслеживание статуса:** Визуальное отображение процесса (ожидание, успех или ошибка).

## 4. Философия дизайна (Cyberpunk Edition)
- **Визуализация:** Темная тема по умолчанию (`CyberBackground` #020617), высококонтрастные неоновые акценты (`CyberCyan`, `CyberAmber`, `CyberEmerald`, `CyberLime`).
- **Типографика:** Использование моноширинных шрифтов `Space Mono` / `IBM Plex Mono` для данных и `Syne` для заголовков.
- **Отклик:** Тактильная отдача (Haptic) при нажатиях; анимация неоновой линии сканирования (`Scanline`) для эффекта "живой системы".

## 5. Протоколы безопасности
- **Подтверждение действий:** Запрет на выполнение деструктивных операций (например, `rm -rf`) без явного подтверждения.
- **Авторизация:** Полная опора на модель разрешений Shizuku. Пользователь должен вручную разрешить доступ.
- **Логирование:** Каждая выполненная команда записывается в `STANDALONE_LOG_STREAM` с меткой времени и кодом завершения.

## 6. Сборка и развертывание
- **Web Build:** `npm run build` генерирует ассеты в папку `dist/`.
- **Capacitor Sync:** `npx cap sync android` переносит файлы в проект Android.
- **Gradle Build:** `./gradlew assembleDebug` создает финальный APK.
